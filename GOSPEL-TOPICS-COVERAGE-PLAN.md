## Revised execution plan (doctrine only)

**Phase 0 — Verify the partials (read-only, ½ day).**
Open each 🟡 page; mark "already sufficient" vs. "needs dedicated page." This will retire
several queue items before any writing.

**Phase 1 — Build the doctrine queue in thematic batches (~30 pages).**
Order suggested by doctrinal foundation, not alphabetics:
Batch 1 (Scripture/Canon) → 2 (Premortal/Heavens) → 3 (Creation/Fall/Mortality) →
4 (Atonement/Justice) → 5 (Priesthood/Ordinances) → 6 (Covenant/Law) →
7 (Discipleship) → 8 (Last Things). ~4 pages per batch keeps quality high; publish each
batch and regenerate the sitemap once per batch.

**Phase 2 — Fold priesthood offices** into the Aaronic/Melchizedek pages as sections.

**Phase 3 (optional) — Promote borderline virtues** (Humility, Peace, Virtue) only if wanted.

### Per-page build checklist
- [ ] Page **name** = slug-safe (hyphens; **no** `?` `/` `#` or spaces) — e.g. `Found-Creation`
- [ ] HTML styled per the ldsdoctrines guide (gold theme, scripture blockquotes)
- [ ] Every doctrine **shown from scripture** — primary refs in blockquotes, linked to
      `churchofjesuschrist.org/...` with `target="_blank" rel="noopener"`
- [ ] `create_page_with_article` → publish → `add_menu_item` under the correct A–Z section
- [ ] Regenerate sitemap once per batch
- [ ] Never `update_article` on a live page (nulls `articlePath`); use `set_article_content`

### Scale
~30 new doctrinal pages + ~26 partial upgrades ≈ **up to 56 pages of real doctrinal work**,
cleanly batched. Everything off-mission is parked in the Out-of-Scope list above.
